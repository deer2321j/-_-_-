项目总览
本项目通过多个子模型独立生成推荐结果，最终采用“双层加权投票”融合形成提交文件。各子模型的训练与预测在其各自目录内完成，融合脚本在项目根目录运行。
数据与输入
复赛数据文件名为"1data.csv"，决赛数据文件名为"111data.csv"。各子模型目录通常已包含所需数据副本。所有脚本均使用相对路径，请在项目根目录运行。
我们过程中的全部中间文件均保存可直接  python FINAL加权.py → 生成最终 submission.csv

-完整复现时，请进入各自对应模型的文件夹内部，各文件所需的环境依赖以及具体步骤我们在readme内部都有写！！！文件名要注意！！！。

- 文件编码建议使用 UTF-8，列名需与脚本一致。

快速生成推荐结果步骤

 在根目录运行：python FINAL加权.py，生成 submissio.csv。

完整复现步骤
- 在项目根目录依次运行（子模型复现需进入到对应目录，完成复现）
1) 分别进入各子模型目录（如 23混推、v5、dspos2、133、f1、v2、决赛classic_autoML），按各目录脚本与说明生成各自 CSV 输出。
2) 返回根目录运行：python 整合rv5到最终投票.py → 生成 七以上的v5.csv。
3) 运行：python Top10加权融合.py → 生成 top10加权输出结果.csv。
4) 运行：python FINAL加权.py → 生成最终 submission-噜啦啦.csv。

注意- 运行位置：默认当前目录为项目根目录，直接按上述相对命令运行。
